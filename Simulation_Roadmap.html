// 01 — OVERVIEW

What does it do?

⚡

RANDOM DELAY

15 – 50 seconds random wait before LED fires

💡

LED TRIGGER

Red LED turns ON — that's your signal to react

⏱️

MILLISECOND ACCURACY

Measures 0 – 9999 ms using 100 MHz clock

🖥️

7-SEG DISPLAY

4-digit result shown with decimal point

The system uses a Finite State Machine (FSM) with 4 states. A free-running counter provides randomness so you can't predict when the LED fires. The result is converted from binary to BCD (Binary Coded Decimal) using the Double Dabble algorithm and displayed on the 4-digit seven-segment display at a refresh rate of ~95 Hz so it appears steady to the eye.

// 04 — ARCHITECTURE

Module breakdown

TOP MODULE

📄 rxn.v — reactionTimer
  • Contains the 4-state FSM
  • Free-running random counter (15–50)
  • Millisecond tick generator
  • Reaction timer (0–9999 ms)
  • Countdown timer (30-bit)
  • Instantiates both sub-modules

BIN2BCD CONVERTER

📄 bin2bcd.v
  • Double Dabble algorithm
  • Converts 14-bit binary → 4 BCD digits
  • Handles 0 – 9999
  • 3-state FSM: idle → op → done
  • Takes 14 clock cycles to convert
  • Fires done_tick when complete

DISPLAY MUX

📄 display.v — displayMuxBasys
  • Multiplexes 4 digits rapidly
  • 18-bit refresh counter
  • ~95 Hz refresh per digit
  • Active-LOW anodes & cathodes
  • Full hex decoder (0–F)
  • Decimal point control per digit

rxn.v — Millisecond tick generator 
// 100 MHz clock → 1ms tick
// 1ms = 100,000 clock cycles → need 17 bits (2^17 = 131072)

assign ms_counter_next =
    (ms_go && ms_counter_reg < 17'd99_999) ? ms_counter_reg + 1'b1 :
    (ms_counter_reg == 17'd99_999)          ? 17'd0              :
                                               ms_counter_reg;

assign ms_tick = (ms_counter_reg == 17'd99_999) ? 1'b1 : 1'b0;

The display shows your result as X.XXX seconds (decimal point after first digit). So 0235 displayed = 0.235 seconds = 235ms. The display can show up to 9999ms (9.999 seconds) before wrapping — plenty of time even for the slowest reactions.

// 09 — DEPLOY

Programming the board

OPTION A — USE BITFILE

Fastest — no synthesis needed
  • Open Vivado Hardware Manager
  • Connect Nexys 4 via USB
  • Click "Open Target → Auto Connect"
  • Click "Program Device"
  • Select reactionTimer_bitfile_.bit
  • Click Program — done!

OPTION B — BUILD FROM SOURCE

Full synthesis in Vivado
  • Create new Vivado project (Nexys 4)
  • Add rxn.v, bin2bcd.v, display.v
  • Set rxn.v as top module
  • Add .xdc constraints file
  • Run Synthesis → Implementation
  • Generate Bitstream → Program
HOW TO PLAY 
// Step 1 — Power on the board
Display shows: 0000   LED: OFF   State: IDLE

// Step 2 — Press BTNU (UP button)
System loads random delay (15–50 seconds worth of cycles)
Display shows: 0000   LED: OFF   State: LOAD
WAIT... don't press anything yet!

// Step 3 — LED turns ON
⚡ REACT NOW! Press BTND (DOWN button) as fast as you can
LED: ON    State: TIMING   ms counter incrementing...

// Step 4 — Your result appears
Display shows your time in milliseconds e.g. 0247 = 247ms
State: W2C   (waiting to clear)

// Step 5 — Press BTNC (CENTRE) to reset and play again
Display: 0000   State: IDLE   Ready for next round!
